正则表达式基础

概念：

regular expression, regex, RE
正则表达式是用来简洁表达一组字符串的表达式，是一种通用的字符串表达框架。
进一步
正则表达式是一种针对字符串表达“简洁”和“特征”思想的工具，可以用来判断某字符串的特征归属。
优势：”一行胜千言“
例如：


用途：

表达文本类型的特征（病毒、入侵等）
同时查找或替换一组字符串
匹配字符串的全部或部分
…
最主要应用在字符串匹配中

使用：

将符合正则表达式语法的字符串转换成正则表达式特征。
一个字符串经过编译后才可以成为一个表达式。p = re.compile(regex)

语法

操作符 说明 实例
. 表示任何单个字符
[] 字符集，表示单个字符的取值范围 [abc]表示a、b、c，[a‐z]表示a到z单个字符
[^] 字符集，表示单个字符的排除范围 [^abc]表示非a或b或c的单个字符
* 前一个字符0次或无限次扩展 abc* 表示ab、abc、abcc、abccc等
+ 前一个字符1次或无限次扩展 abc+ 表示abc、abcc、abccc等
? 前一个字符0次或1次扩展 abc? 表示ab、abc
` ` 左右表达式任意一个
{m} 前一个字符扩展m次 ab{2}c表示abbc
{m,n} 前一个字符扩展[m,n]次 ab{1,2}c表示abc、abbc
^ 匹配字符串开头 ^abc表示abc且在一个字符串的开头
$ 匹配字符串结尾 abc$表示abc且在一个字符串的结尾
() 分组标记，内部只能使用 操作符 (abc)表示abc，`(abc
\d 数字，等价于[0‐9] \d{3}，代表任意3位数
\w 单词字符，等价于[A‐Za‐z0‐9_]包含大小写字母，数字和下划线

Re库基础使用

Re库是Python的标准库，主要用于字符串匹配

表达类型

只是表示，并不是正则表达式。编译后的才称为正则表达式。
re.compile(pattern,flags=0)
raw string类型（原生字符串类型）是，不包含对转义符再次转义的字符串，在字符串前面添加“r”，代表忽视转义
例如：

 r'[1‐9]\d{5}'
r'\d{3}‐\d{8}|\d{4}‐\d{7}'

或者
string类型，即采用“\”来忽视转义字符

'[1‐9]\\d{5}'
'\\d{3}‐\\d{8}|\\d{4}‐\\d{7}'

函数式方法：一次性操作

函数 说明
re.search() 在一个字符串中搜索匹配正则表达式的第一个位置，返回match对象
re.match() 从一个字符串的开始位置起匹配正则表达式，返回match对象
re.findall() 搜索字符串，以列表类型返回全部能匹配的子串
re.split() 将一个字符串按照正则表达式匹配结果进行分割，返回列表类型
re.finditer() 搜索字符串，返回一个匹配结果的迭代类型，每个迭代元素是match对象
re.sub() 在一个字符串中替换所有匹配正则表达式的子串，返回替换后的字符串
注：match对象解析在常用方法后

re.search(pattern, string, flags=0)

在一个字符串中搜索匹配正则表达式的第一个位置，返回match对象
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ string : 待匹配字符串
∙ flags : 正则表达式使用时的控制标记
常用标记 | 说明
–|–|
re.I ， re.IGNORECASE | 忽略正则表达式的大小写，[A‐Z]能够匹配小写字符
re.M ， re.MULTILINE | 正则表达式中的^操作符能够将给定字符串的每行当作匹配开始
re.S， re.DOTALL | 正则表达式中的.操作符能够匹配所有字符（包括换行符），默认匹配除换行外的所有字符

>>> import re
>>> match = re.search(r'[1-9]\d{4}','BIT 100081)
>>> if match: #判断是否为空类型
	print(match.group(0))

10008

re.match(pattern, string, flags=0)

从一个字符串的开始位置起匹配正则表达式，返回match对象
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ string : 待匹配字符串
∙ flags : 正则表达式使用时的控制标记

>>> match = re.match(r'[1-9]\d{5}','BIT 100081')
>>> if match:#防止报错
        	print(match.group(0))
#字符串起始位置，不符合规则，故此时match为空对象，不具备'group'属性
 >>> match.group(0)
Traceback (most recent call last):
  File "<pyshell#25>", line 1, in <module>
    match.group(0)
AttributeError: 'NoneType' object has no attribute 'group'

re.findall(pattern, string, flags=0)

搜索字符串，以列表类型返回全部能匹配的子串
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ string : 待匹配字符串
∙ flags : 正则表达式使用时的控制标记

>>> ls  = re.findall(r'[1-9]\d{5}','BIT100081,BIT100072')
>>> ls
['100081', '100072']

re.split(pattern, string, maxsplit=0, flags=0)

将一个字符串按照正则表达式匹配结果进行分割，返回列表类型。去掉符合规则的部分。
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ string : 待匹配字符串
∙ maxsplit: 最大分割数，剩余部分作为最后一个元素输出。可以控制分割几部分，最后多出来的作为整体输出。
∙ flags : 正则表达式使用时的控制标记

>>> ls  = re.split(r'[1-9]\d{5}','BIT100081,BIT100072')
>>> ls
['BIT', ',BIT', '']#即去掉符合规则的部分
>>> ls  = re.split(r'[1-9]\d{5}','BIT100081,BIT100072',maxsplit=1)
>>> ls
['BIT', ',BIT100072']
>>> ls  = re.split(r'[1-9]\d{5}','BIT100081,BIT100072sdfs',maxsplit=1)
>>> ls
['BIT', ',BIT100072sdfs']

re.finditer(pattern, string, flags=0)

搜索字符串，返回一个匹配结果的迭代类型，每个迭代元素是match对象
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ string : 待匹配字符串
∙ flags : 正则表达式使用时的控制标记

>>> for m in  re.finditer(r'[1-9]\d{5}','BIT100081,AIT100072'):
	if m:
		print(m.group(0))

100081
100072

re.sub(pattern, repl, string, count=0, flags=0)

在一个字符串中替换所有匹配正则表达式的子串，返回替换后的字符串。即替换符合规则的字符串。
∙ pattern : 正则表达式的字符串或原生字符串表示
∙ repl : 替换匹配字符串的字符串
∙ string : 待匹配字符串
∙ count : 匹配的最大替换次数
∙ flags : 正则表达式使用时的控制标记

>>> re.sub(r'[1-9]\d{5}','adb','BIT100081,AIT100072')
'BITadb,AITadb'

Re库的贪婪匹配和最小匹配

Re库默认采用贪婪匹配，即输出匹配长度最长的的子串。

>>> match = re.search(r'PY.*N', 'PYANBNCNDN')
>>> match.group(0)
'PYANBNCNDN

最小匹配，当有操作符可能匹配不同长度字符串时，可以在其后加上‘?’来获得最小匹配结果。
操作符| 说明
–|–|
*? | 前一个字符0次或无限次扩展，最小匹配
+? | 前一个字符1次或无限次扩展，最小匹配
?? | 前一个字符0次或1次扩展，最小匹配
{m,n}? | 扩展前一个字符m至n次（含n），最小匹配

>>> match = re.search(r'PY.*?N', 'PYANBNCNDN')
>>> match.group(0)
'PYAN'
